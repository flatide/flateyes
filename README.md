# flateyes

폐쇄망 리눅스 환경용 초경량 이미지 뷰어 (단일 파일 Python 스크립트).
동봉된 [`fe_embed.py`](#캡처-시점에-png에-주석-심기-fe_embedpy)로는 뷰어 없이도
PNG에 주석을 임베드할 수 있다 (대량 캡처 서비스 연동용, 역시 단일 파일).

eog를 대체하기 위해 만들어졌으며, eog의 두 가지 문제를 동시에 해결한다:

| | eog | flateyes |
|---|---|---|
| 같은 디스플레이에서 반복 실행 | 창이 계속 늘어남 (또는 `--single-instance` 시 아래 문제 발생) | **기존 창 하나가 이미지를 교체 표시** (나란히 비교가 필요하면 `--multi`로 새 창) |
| 여러 사용자 디스플레이에 동시 표시 | 단일 인스턴스 모드에서는 불가 | **디스플레이마다 독립 인스턴스로 동시 표시 가능** |

## 동작 원리

- 인스턴스 식별 키 = `(실행 uid, $DISPLAY)`. 이 키로 리눅스 abstract namespace 유닉스 소켓을 생성한다 (디스크에 파일을 남기지 않으므로 stale socket 문제 없음).
- 같은 디스플레이에서 다시 실행하면: 기존 인스턴스 소켓에 이미지 경로만 전달하고 **즉시 종료** (GTK도 로드하지 않아 수십 ms 수준). 기존 창이 새 이미지로 교체되고 앞으로 올라온다.
- 디스플레이가 다르면 소켓 이름이 달라지므로 각 사용자 디스플레이마다 창이 하나씩 독립적으로 뜬다.
- `--multi`(`-m`) 옵션은 이 단일 인스턴스 로직을 통째로 건너뛴다: 소켓에 접속하지도, 소유하지도 않고 항상 독립된 새 창을 연다 (이미지 나란히 비교 등).

## 요구 사항

추가 패키지 설치 불필요 — GNOME(eog)이 있는 시스템에 기본 포함된 것만 사용:

- Python 3
- PyGObject + GTK3 (`python3-gobject`, RHEL/CentOS GNOME 환경 기본 포함)

대상 서버에서 사전 확인:

```sh
python3 -c 'import gi; gi.require_version("Gtk", "3.0"); print("OK")'
```

`OK`가 출력되면 바로 사용 가능. 실패한다면(파이썬 바인딩이 없는 호스트)
아래 **포터블 번들**로 실행한다.

## 설치

파일 하나만 복사하면 된다:

```sh
cp flateyes.py /usr/local/bin/flateyes   # 경로는 자유
chmod +x /usr/local/bin/flateyes
```

### PyGObject가 없는 호스트: 포터블 번들

eog는 실행되지만 `python3-gobject`가 없는(위 사전 확인이 실패하는) 오래된
최소 설치 호스트는, Python + PyGObject + GTK3 전체를 자체 포함하는 번들로
실행한다. 시스템에는 아무것도 설치하지 않으며, X 디스플레이와 한글 시스템
폰트만 가져다 쓴다.

```sh
# 인터넷이 되는 PC(맥/리눅스)에서 번들 생성 (~87MB tar)
./make_portable.sh

# 대상 호스트(x86_64, glibc 2.17+ = RHEL/CentOS 7 이상)에서
tar xzf flateyes-portable-*.tar.gz -C /opt    # 위치는 자유
/opt/flateyes-portable/selfcheck              # 창 없이 스택 검증
/opt/flateyes-portable/flateyes image.png
```

- 기능·옵션·단일 인스턴스 동작은 일반 실행과 완전히 동일하다.
  `ln -s /opt/flateyes-portable/flateyes /usr/local/bin/flateyes`로 링크해 쓰면 된다.
- 첫 실행 시 `~/.cache/flateyes-rt/`에 이 호스트용 캐시(스키마·이미지 로더·폰트)를
  자동 생성한다. 이후 flateyes.py 업데이트는 번들 폴더 안의 `flateyes.py`만
  교체하면 된다 (런타임 재사용).
- RHEL6(glibc 2.12)은 지원 불가 — `selfcheck`가 즉시 판별해 준다. 상세한 설치·문제
  해결 안내는 번들 안의 `README-PORTABLE.txt` 참조.

## 사용법

```sh
DISPLAY=:1 flateyes /path/to/image.jpg

# 폴더를 주면(생략하면 현재 폴더) 썸네일 브라우저가 열려
# 그 폴더의 이미지와 하위 폴더를 탐색할 수 있다
DISPLAY=:1 flateyes /path/to/folder
DISPLAY=:1 flateyes

# 범례(legend) 이미지를 우하단에 오버레이로 함께 표시
DISPLAY=:1 flateyes --legend /path/to/legend.png /path/to/image.jpg

# ruler 환산 배율 지정: 3.2 픽셀 = 1 um
DISPLAY=:1 flateyes --ppu 3.2 /path/to/image.jpg

# 열면서 주석을 함께 그리기 (박스·타원·선·측정·텍스트, 반복 가능)
DISPLAY=:1 flateyes --box 120,80,360,240,red \
                    --ruler 100,300,400,300 \
                    --text "130,250,defect A" /path/to/image.png

# 이미 뷰어가 떠 있어도 기존 창을 건드리지 않고 독립된 새 창으로 열기
# (두 이미지를 나란히 놓고 비교하는 용도, -m 축약형)
DISPLAY=:1 flateyes --multi /path/to/other.png

# 다중 배율 스택(mip-map 방식) 표시
DISPLAY=:1 flateyes --stack /path/to/site.tds

# 스택을 manifest 파일 없이 명령줄로 직접 구성
DISPLAY=:1 flateyes --level site_5x.png --ppu 0.8 \
                    --level site_20x.png --ppu 3.2 \
                    --level site_50x.png --ppu 8.0
```

- 첫 실행: 해당 디스플레이에 창을 열고 계속 떠 있음 (백그라운드 상주).
- 이후 실행: 같은 창에 이미지를 교체 표시하고 즉시 종료 (exit 0).
- 이미지 로드 실패 시: 오류를 stderr로 출력하고 exit 1 (기존 창의 이미지는 유지).
- `-m` / `--multi`: 위의 단일 인스턴스 동작을 건너뛰고 항상 독립된 새 창을
  연다 (창이 닫힐 때까지 상주). 이 창은 요청 수신 소켓을 만들지 않으므로
  기본 인스턴스와 충돌하지 않으며, 이후 `--multi` 없는 실행은 여전히 기본
  창으로만 전달된다. 모든 옵션(주석·스택 등)을 동일하게 쓸 수 있다.

### Legend 오버레이

`-l` / `--legend LEGEND_FILE` 옵션을 주면 범례 이미지가 본 이미지 우하단에 오버레이로 표시된다.

- 범례는 창의 최대 1/3 크기로 축소 표시된다 (확대는 안 함). 창 크기를 바꾸면 따라서 조절된다.
- 요청 단위로 상태가 정해진다: 이후 실행에서 `--legend`를 주면 범례도 교체되고, 주지 않으면 범례가 사라진다.
- `i` 키로 상단 키 안내와 함께 숨기거나 다시 표시할 수 있다.
- 범례 위에서의 클릭/휠도 본 이미지에 전달된다 (드래그 패닝, Ctrl+휠 확대 모두 동작).

### Ruler (거리 측정)

`r` 키로 측정 모드를 켜면 커서가 십자선으로 바뀐다.

- 클릭으로 시작점을 찍으면 마우스 위치까지의 가이드선과 거리가 실시간 표시되고, 두 번째 클릭으로 확정된다. 포인트는 버튼에서 손을 뗄 때 찍힌다.
- 측정 모드 중에도 **드래그 패닝이 그대로 동작한다**: 드래그(4px 이상 이동)하면 포인트를 찍지 않고 화면을 이동한다.
- 기본은 **수평/수직 스냅**: 마우스 이동량이 큰 축 방향으로만 가이드선이 나간다. `Shift`를 누르고 있으면 자유 방향으로 측정한다.
- `Ctrl`을 누르고 있으면 **지금 진행 중인 방향이 고정**된다: 스냅 중에는 수평/수직 축이 고정되어 45°를 넘게 움직여도 축이 뒤집히지 않고, `Shift` 자유 방향 중에는 현재 각도가 고정되어 마우스를 어디로 움직여도 길이만 그 방향을 따라 변한다. `Ctrl`을 놓으면 다시 마우스를 따라간다.
- 측정점은 이미지 픽셀에 고정되므로 확대/스크롤해도 측정선이 이미지를 따라간다. 거리도 화면 배율과 무관하게 원본 픽셀 기준이다.
- **엣지 스냅** (`m` 토글, 기본 꺼짐): 켜면 측정점이 커서 주변(화면 기준 약 14px)에서
  밝기가 가장 급하게 변하는 경계로 끌려가 붙는다. 마우스 포인터 자체는 움직이지
  않고, 첫 점을 찍기 전에는 십자 마커가 스냅될 위치를 미리 보여준다. 끝점은
  **측정 진행 방향의 성분만** 스냅되므로 측정선과 나란한 경계에 옆으로 끌려가지
  않고, 수평/수직/각도 고정 상태에서는 점이 측정선을 벗어나지 않는다. 스냅
  위치는 경계의 서브픽셀 중심으로 보간되어 픽셀 격자보다 정밀하다. 주변에 뚜렷한
  경계가 없으면(평탄한 영역) 스냅하지 않으며, `Alt`를 누르고 있는 동안은 스냅이
  일시 해제된다.
- 두 번째 클릭으로 확정된 측정은 **주석으로 보존된다**: 여러 측정을 동시에 유지할
  수 있고, 측정 모드를 나가거나 확대/스크롤해도 그대로 남는다. `u`/`y`
  (실행 취소/다시 실행)와 저장·복원에 도형과 동일하게 포함된다.
  거리 표시는 현재 PPU로 렌더링 시점에 환산되므로 PPU를 바꾸면 함께 갱신된다.
  단, 측정은 일시적 작업 보조로 취급되어 **측정만 바뀐 상태로 종료하거나 다른
  이미지로 이동할 때는 저장 여부를 묻지 않는다** (도형·텍스트·PPU 변경이 섞여
  있으면 묻는다. `Ctrl+S`로 저장하면 측정도 함께 저장된다).
- 측정이 여러 개면 거리 라벨끼리 겹치지 않도록 자동으로 벌려 배치되고, 각 라벨은
  점선 리더로 자기 측정선에 연결된다. 십자로 교차하는 측정이라도 어느 라벨이 어느
  선의 값인지 구별된다. 측정이 하나뿐이면 리더 없이 측정선 중앙 옆에 표시된다.
- 확정 후 다시 클릭하면 새 측정이 시작된다. `Esc` 또는 `r`로 측정 모드를 종료한다.
- **PPU(pixels per unit)**: `--ppu` 옵션 또는 `p` 키 대화상자로 "1 단위 = 몇 픽셀"을 입력하면 `12.34 um  (307 px)` 형태로 환산 표시된다. 단위 이름은 `--unit`으로 변경 (기본 `um`). PPU를 지정하지 않으면 픽셀 거리만 표시된다.
- **PPU는 파일별로 저장된다**: `p` 대화상자로 입력한 PPU/단위는 `Ctrl+S`로
  저장할 때 그 파일의 주석 메타데이터에 함께 기록되어 같은 파일을 다시
  열면(폴더 브라우징 포함) 복원된다. 대화상자에서 빈 값으로 확인한 뒤
  저장하면 기록된 값도 지워진다. 저장된 값이 없는 파일에는
  창에 유지되던 마지막 값(sticky)이 쓰이고, 요청에 `--ppu`가 명시되면 그것이
  저장된 값보다 우선한다(저장하지는 않음).

### 도형·텍스트 주석

이미지 위에 표시용 주석을 얹을 수 있다. 주석은 이미지 좌표(스택은 um 좌표)에
고정되어 확대/스크롤/레벨 전환을 그대로 따라간다.

- `d`(도형 그리기): 대화상자에서 도형 종류(box·ellipse·line)와 스타일을 함께
  고르고 확인하면 그리기 모드가 시작된다. 스타일은 outline 색(윤곽선 — line
  도형을 고르면 행 라벨이 "line"으로 바뀌어 라인 색이 된다)과 fill
  색(박스·타원 내부)을 공통 팔레트(black·white·red·orange·green·sky·pink,
  색 견본 표시)에서 고르고, 각 행의 "use" 체크로 윤곽선/채우기 사용 여부를,
  "opaque" 체크로 채우기의 불투명/반투명(0.35)을 정한다. "black halo" 체크는
  윤곽선·라인을 둘러싸는 검은 테두리 여부다 (색이 배경에 묻히지 않게 하는
  용도, 기본 켜짐). 도형이 안 보이는
  조합을 막기 위해 윤곽선과 채우기 중 마지막 남은 하나는 끌 수 없고, 선(line)은
  항상 라인 색으로 그려진다. 굵기("width" 1~8px)와 선 종류(solid/dashed/dotted)는
  라인과 박스·타원 윤곽선에 공통으로 적용된다. 선택은 이후 그리는 도형에
  적용된다 (기존 주석은 유지, 마지막 설정 기억).
  그리기는 클릭 두 번으로 두 모서리(라인은 양 끝점)를 지정한다 (포인트는
  버튼을 뗄 때 찍히고, 드래그는 패닝). 첫 클릭 후 미리보기가 표시되며,
  `Shift`를 누르면 박스·타원은 정사각형/정원, 라인은 수평/수직/45°로 고정된다.
  모드 종료는 `Esc`.
- `t`(텍스트): 클릭한 위치에 대화상자로 텍스트와 폰트 크기(6~96pt)를 입력한다.
  여러 줄 입력 가능 — `Enter`는 줄바꿈, `Ctrl+Enter`가 확인. 글자 색("text")과
  배경("background" 사용 여부·색·"opaque" 불투명 여부)을 대화상자 안의 콤보로
  직접 선택하며, 도형(`d`) 설정과는 별개로 유지된다. 폰트 크기·색·배경 설정은
  입력 중인 텍스트에 실시간 반영된다. 마지막 사용한 크기·색·배경 설정이
  기억된다. 텍스트 크기는 확대해도 화면 기준으로 일정하다. 대화상자 라벨은
  한글 글꼴이 없는 X 서버(XQuartz 등)에서도 깨지지 않도록 영어로 표기한다.
  대화상자에서 `Esc`는 입력만 취소하고 텍스트 모드는 유지된다(다시 클릭해
  입력 가능) — 모드 종료는 화면에서 `Esc` 또는 `t`.
- **한글 입력**: 시스템 입력기(ibus 등)가 없는 환경을 위해 두벌식 한글 조합기를
  내장했다. 대화상자에서 `Shift+Space`(또는 한/영 키, 체크박스)로 한글 모드를
  켜면 IME 없이 한글이 입력된다. 시스템 입력기가 동작하는 환경에서는 켜지 않으면
  아무 개입도 하지 않는다.
- 모드에 따라 커서가 바뀐다: ruler 십자선, 도형 셀(+), 텍스트 I-빔. 드래그 패닝 중에는 잡는 손 모양.
- 모드가 켜져 있는 동안에는 하단 토스트에 현재 모드가 계속 표시된다 —
  도형이면 색 견본·굵기·선 종류·채우기까지 요약해서 보여준다. 주석 선택(`s`)의
  "selected i/n" 안내와 크기 조절(`e`)의 "resize" 안내도 같은 방식으로 유지된다.
  저장 알림 같은 일시적 토스트가 잠깐 덮어도 사라진 뒤 모드 표시로 되돌아오며,
  `Tab`으로 모든 표시를 숨기면 토스트도 함께 숨는다.
- `s`: 그려진 주석 선택 — 누를 때마다 최신 주석부터 순회하고(`Shift+s`는 반대
  방향) 선택된 주석에 노란 모서리 핸들이 표시된다 (화면 밖에 있으면 자동으로
  스크롤해 보여준다). 선택 중에는 방향키가 화면 이동 대신 주석을 화면 기준
  1픽셀씩 옮기고(`Shift`+방향키는 10픽셀, 이미지 밖으로는 못 나감),
  `Delete`/`BackSpace`가 주석을 삭제하며, `Esc`가 선택을 해제한다. 이동과
  삭제도 `u`/`y` 실행 취소에 포함된다 (연속 이동은 한 번으로 묶인다).
  도형·측정 모드 키를 누르면 선택은 해제된다.
- 선택 중 `e`: 주석 수정 — 도형·선·측정은 누를 때마다 방향키가 움직이는
  대상이 순환한다 (전체 이동 → 모서리/끝점 B → A → 전체 이동, 활성 앵커는
  흰색 핸들로 표시). 앵커가 켜진 동안 방향키는 그 점만 끌어 크기·길이·각도를
  조절하고, `Esc`는 전체 이동으로 한 단계 돌아온다. 텍스트는 기존
  내용·크기·색·배경이 채워진 입력 대화상자가 다시 열려 세밀하게 수정할 수
  있다. 수정도 `u`/`y`로 되돌릴 수 있다.
- `u`/`y`: 실행 취소/다시 실행 — `u`가 최신 주석부터 하나씩 지우고 `y`가 되살린다.
  파일에서 복원된 주석도 똑같이 지울 수 있다. 모드 종료는 `Esc`
  (`t`/`r`은 재입력으로도 종료).
- ruler와 주석 모드는 상호 배타적이며(하나를 켜면 다른 것이 꺼짐), `Tab` 드로잉
  토글에 포함된다.
- **명시적 저장(`Ctrl+S`)**: 주석과 PPU/단위는 자동 저장되지 않는다 —
  메모리에만 있다가 `Ctrl+S`를 눌렀을 때 기록된다. 저장 안 된 변경이 있으면
  창 제목의 파일명 앞에 `*`가 붙고(실행 취소로 저장 시점까지 되돌리면
  사라짐), `q`·창 닫기·다른 이미지로 이동(`,`/`.`, 썸네일 브라우저) 때
  저장할지 묻는 대화상자(Save/Discard/Cancel)가 뜬다. 측정(ruler)만 바뀐
  상태는 예외로, `*` 표시도 질문도 없이 조용히 넘어간다. 또한 외부 실행이
  같은 창에 새 이미지를 요청해 교체될 때는 묻지 않으며 변경은 사라진다.
  저장 위치는 포맷에 따라 다르다:
  - **PNG**: 이미지 파일 안에 `iTXt` 청크로 내장한다 — 사이드카를 만들지
    않는다. 원본 파일은 이때만 수정되고(임시 파일에 쓴 뒤 원자적 교체,
    픽셀은 그대로) 다른 뷰어에서는 평범한 PNG로 열리며, 파일을 복사하면
    주석이 함께 이동한다. 구버전이 남긴 `.fe` 사이드카는 청크가 없는 동안만
    읽어오며 `Ctrl+S` 때 정리된다.
  - **그 외 포맷**: 이미지 옆 사이드카 파일 `<이미지경로>.fe`(텍스트
    key=value 형식)에 저장한다 (스택은 manifest 경로 기준). 이미지
    디렉터리가 읽기 전용이면 `~/.cache/flateyes/`에 대신 저장한다.

  같은 파일을 다시 열면 저장된 주석이 복원되고, 주석 없이 저장하면 내장
  청크/사이드카를 제거한다.

### 노트 (이미지 단위 메모)

`n` 키로 이미지 전체에 대한 자유 텍스트 메모를 입력/수정한다. 텍스트 주석과
달리 위치·크기·색상이 없으며, 좌상단에 키 안내·상태 표시와 같은 정보 칩
형태로 고정 표시된다 (legend와 겹치지 않도록 왼쪽에, legend가 하단 경로
표시를 피하는 것처럼 키 안내 아래로 약간 내려서).

- 입력 대화상자는 텍스트 주석과 같은 방식: `Enter` 줄바꿈, `Ctrl+Enter`
  확인, `Esc` 취소, 한글 내장 입력(Hangul 체크, `Shift+Space` 토글).
- 내용을 모두 지우고 OK 하면 노트가 제거된다. 노트는 실행 취소(`u`)의
  대상이 아니다 — 다시 `n`으로 고치면 된다.
- `i`/`Tab`의 정보 표시 숨김을 함께 따르고, `Ctrl+C` 화면 복사에서는 다른
  정보 표시처럼 자동 제외된다.
- 저장은 주석과 동일하게 `Ctrl+S` — PNG는 파일 안에(`note=` 줄), 그 외
  포맷은 `.fe` 사이드카에. 파일을 다시 열면 자동 복원된다.
- 캡처 서비스에서 fe_embed.py로도 넣을 수 있다: CLI `--note TEXT`,
  파이썬 `fe.embed(..., note="...")`. 뷰어를 띄우면서 지정하려면
  `flateyes --note TEXT image.png` (미저장 상태로 표시).

### 실행 옵션으로 주석 추가

이미지를 열면서 명령줄 옵션으로 주석을 함께 그릴 수 있다. 외부 프로그램이
결함 좌표 등을 표시한 채로 뷰어를 띄우는 용도이며, 각 옵션은 여러 번
반복할 수 있다.

```sh
flateyes --box X1,Y1,X2,Y2[,COLOR[,FILL[,WIDTH[,DASH]]]] image.png
flateyes --ellipse X1,Y1,X2,Y2[,COLOR[,FILL[,WIDTH[,DASH]]]] image.png
flateyes --line X1,Y1,X2,Y2[,COLOR[,WIDTH[,DASH]]] image.png
flateyes --ruler X1,Y1,X2,Y2 image.png
flateyes --text X,Y[,STYLE...],TEXT image.png
flateyes --note TEXT image.png
flateyes --json annos.json image.png
```

- **좌표**는 이미지 픽셀 기준 (스택은 ruler와 같은 um 세계 좌표).
- **COLOR**: 팔레트 이름(black·white·red·orange·green·sky·pink) 또는
  `#RRGGBB`. 생략하면 red. `0`이면 윤곽선 없음 (FILL 필수).
- **FILL**(박스·타원 내부): 색 이름/`#RRGGBB`는 반투명(0.35),
  `#RRGGBBAA`로 주면 AA가 `80` 이상일 때 불투명. 생략하면 채우기 없음.
- **WIDTH**: 굵기 1~8px (기본 1). **DASH**: `solid`/`dashed`/`dotted`
  (또는 0/1/2, 기본 solid).
- `--ruler`는 확정된 측정으로 추가되어 현재 PPU/단위로 환산 표시된다
  (`--ppu`와 조합 가능).
- `--text`의 STYLE 자리에는 `size=`(6~96), `color=`(COLOR와 동일 규칙),
  `bg=`(배경)를 쉼표로 이어 넣을 수 있다 — 뷰어의 `t` 대화상자가 제공하는
  스타일 전부다. 생략하면 16pt·red·반투명 검정 배경. `bg=`는 `0`(배경
  없음), 색 이름/`#RRGGBB`(반투명), `#RRGGBBAA`(AA가 `80` 이상이면
  불투명)를 받는다.

  ```sh
  flateyes --text '120,60,size=20,color=white,bg=#000000FF,DEFECT #17' image.png
  ```

  TEXT는 스타일 필드가 아닌 첫 필드부터 끝까지 전부이며 쉼표를 포함할 수
  있고 리터럴 `\n`이 줄바꿈이 된다. 내용이 하필 `size=` 등으로 시작한다면
  `text=`를 붙여 시작점을 명시한다 (`--text '10,10,text=size=6 메모'`).
- `--note`는 `n` 키의 노트를 실행 시점에 지정한다 — 파일에 저장된 노트가
  있으면 화면에서 대체하며, 다른 DRAW 옵션과 마찬가지로 `Ctrl+S` 전까지는
  미저장 상태다. 리터럴 `\n`이 줄바꿈이 된다.
- `--json`은 주석 여러 개를 JSON 파일로 한 번에 받는다 (`-`는 표준
  입력, 반복 가능). 형식은 아래 fe_embed.py의 것과 완전히 같다 — 배열이면
  주석 목록, 객체면 `ppu`/`unit`/`note`까지 함께 적용된다 (명시한
  `-p`/`-u`/`--note`가 JSON 값보다 우선). 주석은 옵션이 놓인 위치
  순서대로 다른 DRAW 옵션과 함께 적용된다.
- 이미 실행 중인 창으로 전달(포워딩)될 때도 동일하게 적용되며, 스택
  (`--stack`/`--level`)과도 함께 쓸 수 있다. 폴더 인자와는 함께 쓸 수 없다.
- 추가된 주석은 손으로 그린 것과 똑같이 동작한다: 파일에 저장돼 있던 주석
  위에 얹히고, 선택(`s`)·수정(`e`)·삭제·실행 취소(`u`)가 되며, **저장되지
  않은 상태**로 열리므로(제목에 `*`) 유지하려면 `Ctrl+S`로 저장해야 한다.

### 캡처 시점에 PNG에 주석 심기 (fe_embed.py)

뷰어를 띄우지 않고도 PNG 파일에 주석을 임베드할 수 있는 독립 유틸리티.
대량 캡처 서비스가 캡처와 동시에 결함 박스·텍스트 등을 파일 안에 넣어 두면,
flateyes로 여는 순간 그대로 표시되고 `Ctrl+S` 편집도 같은 청크에 이어진다.

- flateyes와 동일한 방식: 픽셀은 건드리지 않고 `iTXt` 청크(keyword
  `flateyes`)만 IEND 앞에 삽입하므로 다른 프로그램에서는 평범한 PNG다.
- 표준 라이브러리만 사용 — `fe_embed.py` 한 파일만 복사하면 어디서든 동작
  (뷰어와 달리 GTK도 필요 없다).
- 좌표는 이미지 픽셀 기준. 옵션 문법(`--box` 등)과 색·굵기·점선 규칙은
  위의 "실행 옵션으로 주석 추가"와 동일하다.

```sh
# CLI — 여러 파일에 같은 주석을 일괄 임베드
python3 fe_embed.py --box 120,80,420,300,red,0,2 \
                    --text '120,60,size=20,color=white,DEFECT #17' \
                    shot_*.png

# JSON으로 대량 지정 ("-"는 stdin), PPU(측정 배율)·노트도 함께 저장
python3 fe_embed.py --json annos.json --ppu 8.5 --unit um \
                    --note '캡처 장비 3호기' shot_0001.png

# 확인 / 기존 주석 유지하며 추가 / 제거
python3 fe_embed.py --dump shot_0001.png
python3 fe_embed.py --append --line 0,0,640,480,green shot_0001.png
python3 fe_embed.py --strip shot_0001.png
```

JSON은 두 형식을 받는다. **배열**이면 주석 목록이고, **객체**면 ppu·unit·
note까지 파일 하나에 담을 수 있다 (모든 키 선택, `--ppu`/`--unit`/`--note`를
명시하면 그 값이 JSON보다 우선). 완전한 샘플:

```json
{
  "ppu": 8.5,
  "unit": "um",
  "note": "캡처 장비 3호기\n조도 재확인",
  "annotations": [
    {"kind": "box", "x1": 120, "y1": 80, "x2": 420, "y2": 300,
     "color": "red", "width": 2, "dash": "dashed"},
    {"kind": "box", "x1": 40, "y1": 40, "x2": 90, "y2": 90,
     "outline": false, "fill": "orange", "fill_opaque": true},
    {"kind": "ellipse", "a": [500, 100], "b": [640, 200],
     "color": "#35C5FF", "fill": "sky", "casing": false},
    {"kind": "line", "x1": 0, "y1": 0, "x2": 300, "y2": 200,
     "color": "green", "width": 4, "dash": "dotted"},
    {"kind": "ruler", "x1": 100, "y1": 500, "x2": 400, "y2": 500},
    {"kind": "text", "x": 120, "y": 50, "text": "DEFECT #17",
     "size": 20, "color": "white", "bg_color": "black", "bg_opaque": true},
    {"kind": "text", "at": [500, 220], "text": "배경 없는 라벨", "bg": false}
  ]
}
```

필드 레퍼런스 (기울임 = 생략 가능, 색은 팔레트 이름
black·white·red·orange·green·sky·pink 또는 `#RRGGBB`):

| kind | 필수 | 선택 |
|---|---|---|
| 최상위 객체 | — | *ppu*(>0, ruler 환산 배율), *unit*(기본 um), *note*(이미지 메모, `\n` 줄바꿈), *annotations*(배열) |
| `box` / `ellipse` | 좌표 | *color*(기본 red), *width*(1~8, 기본 1), *dash*(`solid`/`dashed`/`dotted`, 기본 solid), *casing*(검은 테두리, 기본 true), *fill*(채움 색, 기본 없음), *fill_opaque*(기본 false=반투명), *outline*(false면 윤곽선 없음 — fill 필수) |
| `line` | 좌표 | *color*, *width*, *dash*, *casing* (box와 동일) |
| `ruler` | 좌표 | 없음 (ppu/unit로 환산 표시) |
| `text` | `text`, 위치 | *size*(6~96, 기본 16), *color*(기본 red), *bg*(배경 사용, 기본 true), *bg_color*(기본 black), *bg_opaque*(기본 false=반투명 0.35) |

- 좌표: 도형·ruler는 `"x1"/"y1"/"x2"/"y2"` 또는 `"a": [x, y], "b": [x, y]`,
  텍스트는 `"x"/"y"` 또는 `"at": [x, y]` — 모두 이미지 픽셀 기준.
- 같은 파일을 `flateyes --json`에 줘도 동일하게 해석된다 (뷰어에서는
  ppu가 `-p`, note가 `--note`에 해당하는 값으로 적용되고 `Ctrl+S` 전까지
  미저장).

파이썬 서비스라면 모듈로 직접 사용한다 (파일 저장 전 메모리 단계에서
`embed_bytes`로 처리 가능):

```python
import fe_embed as fe

fe.embed("shot_0001.png", [
    fe.box(120, 80, 420, 300, color="red", width=2),
    fe.text(120, 60, "DEFECT #17", size=20, color="white"),
], ppu=8.5, unit="um", note="캡처 장비 3호기")

blob = fe.embed_bytes(png_bytes, annos)   # bytes -> bytes
annos, ppu, unit, note = fe.read("shot_0001.png")
```

`python3 fe_embed.py --selftest`를 flateyes.py 옆에서 실행하면 모든 주석
종류를 양쪽 구현으로 교차 왕복시켜 포맷 호환을 검증한다 (둘 중 한쪽의
포맷을 수정했다면 반드시 실행).

### 썸네일 브라우저

`b` 키로(또는 폴더를 인자로 실행하면 처음부터) 썸네일 브라우저가 열린다.
한 번에 한 폴더만 읽으므로 하위 폴더가 아무리 깊어도 느려지지 않는다.

- 현재 폴더의 하위 폴더(폴더 아이콘)와 이미지(썸네일)가 나열된다. 썸네일은
  화면을 막지 않도록 백그라운드에서 여러 장씩 병렬로 만들어지고, 화면에 보이는
  항목부터 채워진다.
- 만든 썸네일은 표준 썸네일 캐시(`~/.cache/thumbnails/`)에 저장해서 같은 폴더를
  다시 열면 즉시 표시된다. eog·노틸러스가 이미 만들어 둔 썸네일도 그대로
  재사용한다 (원본 파일이 바뀌면 자동으로 다시 만든다).
- 이미지를 보는 동안에도 그 폴더의 썸네일 캐시가 백그라운드(낮은 우선순위)에서
  미리 만들어진다 — 나중에 `b`를 누르면 전부 즉시 뜬다. 브라우저가 열려 있는
  동안에는 화면 채우기가 우선이라 백그라운드 작업은 멈춘다.
- 내용이 클라우드에만 있는 자리표시자 파일(iCloud 최적화 저장소 등 — 읽는 순간
  다운로드가 시작되는 파일)은 구분해서 처리한다: 브라우저에서는 로컬 파일의
  썸네일을 전부 먼저 만들고 자리표시자는 마지막에 처리하며, 백그라운드 캐시
  생성은 자리표시자를 아예 건너뛴다 (폴더 전체가 일괄 다운로드되는 것을 방지).
- 방향키로 이동하고 `Enter`(또는 더블클릭)로 연다 — 폴더는 그 폴더로 들어가고,
  이미지는 뷰어 화면으로 연다. `..` 항목 또는 `BackSpace`로 상위 폴더로 이동.
- `Esc` 또는 `b`: 보던 이미지로 복귀 (이미지 없이 폴더로 시작한 직후에는 무시).
  `q`: 종료. 하단에 현재 폴더 경로와 폴더/이미지 개수, 키 안내가 표시된다.
- 브라우저에서 연 이미지를 보는 중에는 `Esc`(또는 `b`)로 다시 브라우저로 돌아간다
  — 그 이미지가 선택된 채로 열린다. 취소할 선택·측정·도구 모드가 있으면 그것부터
  빠져나온 뒤 다음 `Esc`에서 브라우저로 돌아간다.
- 저장 안 된 주석이 있는 상태에서 다른 이미지를 열면 저장 여부를 묻는다
  (Save/Discard/Cancel — 취소하면 브라우저에 그대로 남는다. 측정만 바뀐
  경우에는 묻지 않는다).
- 뷰어로 돌아온 뒤의 `,`/`.` 이동은 그 이미지의 폴더 기준으로 동작한다.
  스택 모드에서는 `,`/`.`와 마찬가지로 비활성.

### 다중 배율 스택 (mip-map)

같은 지점을 여러 배율로 촬영한 이미지들을 하나의 세트로 보여준다. 확대할수록 더
높은 배율(더 높은 PPU)의 이미지로 자동 전환되어 확대 정밀도가 올라간다.

두 가지 방법으로 구성한다: `--stack`(`-s`)에 manifest 파일을 넘기거나, `--level IMG --ppu N [--center X,Y]`를 반복해서 명령줄로 직접 지정한다 (`--ppu`/`--center`는 직전 `--level`에 적용). manifest 파일은 한 줄에 `key=value` 하나씩:

```
# site.tds — 배율 순서는 무관 (ppu 기준으로 자동 정렬)
unit=um
level=site_5x.png      # 경로는 이 파일 위치 기준 상대경로 가능
ppu=0.8                # 이 레벨의 pixels per um
level=site_20x.png
ppu=3.2
level=site_50x.png
ppu=8.0
center=1.5,-2.0        # 선택: 촬영 중심이 미세하게 어긋난 경우 보정(um)
```

- 모든 레벨은 같은 중심을 공유한다고 가정하며(`center=`로 레벨별 보정 가능), 내부적으로 um 기준 공통 좌표계로 정합된다.
- **레벨 자동 전환**: 확대/축소 시 뷰포트를 커버하면서 업스케일이 필요 없는 가장 낮은 배율 레벨을 선택한다. 전환 시 화면 중앙의 지점이 유지된다. 확대 위치가 고배율 이미지의 커버리지 밖(가장자리)이면 저배율 이미지를 계속 쓴다.
- **다음 레벨 영역 표시**: 한 단계 높은 배율 레벨이 커버하는 영역이 하늘색 외곽선으로 표시된다 (최고 배율 레벨에서는 표시 없음). 외곽선 안쪽으로 확대하면 자동 전환된다. `o` 키로 토글할 수 있다.
- 실효 최대 확대율은 "최고 배율 레벨의 200%"까지 커진다.
- `[` / `]` 키로 이전/다음 배율 레벨의 100% 크기로 바로 이동할 수 있다.
- **Ruler 연동**: 측정점이 um 좌표에 고정되므로 측정 도중 레벨이 바뀌어도 유지되고, 환산은 manifest의 ppu가 자동 적용된다 (`p` 입력 불필요). 좌하단 상태 표시에 활성 레벨 번호/파일/배율/PPU가 표시된다.
- 레벨 이미지는 요청 시 전부 디코드된다 (2000x2000 3레벨 기준 약 50MB). 하나라도 실패하면 기존 화면이 유지된다.

### 런처(기존 eog 실행 프로그램)에서 호출

eog를 호출하던 자리에 그대로 치환하면 된다. 첫 실행 프로세스는 창이 닫힐 때까지 살아 있으므로, 런처가 종료를 기다리지 않도록 분리 실행 권장:

```sh
sudo -u tduser DISPLAY=:1 XAUTHORITY=/home/tduser/.Xauthority \
    setsid /usr/local/bin/flateyes /data/images/alarm.png >/dev/null 2>&1 &
```

같은 명령을 몇 번을 실행해도 창은 디스플레이당 1개만 유지된다. 두 번째 실행부터는 이미 떠 있는 인스턴스에 경로만 넘기고 바로 끝나므로 `&` 없이 호출해도 무방하다.

## 키 조작

창 상단에 사용 가능한 키 안내가, 좌하단에 배율·해상도·PPU(스택은 레벨 포함) 상태가,
우하단에 파일 전체 경로가 표시된다 (매우 긴 경로는 중간 생략, 마우스를 올리면
툴팁으로 전체 표시). `i`로 legend와 함께 숨김/복원.

| 키 | 동작 |
|---|---|
| `+` / `-` (Ctrl+휠) | 확대 / 축소 (최대 200%) |
| `0` | 원본 크기 (100%) |
| `f` | 창 크기에 맞춤 (기본값, 축소만 하고 확대는 안 함) |
| `Enter` / `F11` | 전체 화면 토글 (원격 환경에서는 F11이 클라이언트에 가로채일 수 있어 `Enter` 권장) |
| 방향키 / 휠 / 마우스 드래그 | 스크롤(패닝) |
| `,` / `.` | 같은 폴더의 이전/다음 이미지 열기 (이름 자연 정렬 순서, 끝에서 순환. 스택 모드에서는 비활성) |
| `b` | 썸네일 브라우저 — 현재 폴더의 하위 폴더와 이미지를 썸네일로 표시하고 탐색한다 (`Enter`/더블클릭 열기, `BackSpace`/`..` 상위 폴더, `Esc`/`b` 복귀, 스택 모드에서는 비활성). 폴더를 인자로 실행하면 이 화면으로 시작한다. 브라우저에서 연 이미지를 보는 중에는 `Esc`(또는 `b`)로 그 이미지가 선택된 브라우저로 되돌아간다 |
| `i` | 정보 표시(상단 키 안내·좌하단 상태·legend·다음 레벨 외곽선·좌상단 노트) 숨김/복원 — `o` 개별 상태는 유지 |
| `Tab` | 모든 표시(측정·주석 + 정보 + 토스트) 숨김/복원 — 하나라도 보이면 전부 숨긴다. 숨긴 동안에는 알림 토스트도 뜨지 않는다. 도구 시작 시 드로잉 표시는 자동 복원 |
| `o` | 다음 레벨 영역 외곽선 표시/숨김 |
| `r` | ruler 측정 모드 시작/종료 (`Shift` 자유 방향, `Ctrl` 진행 중인 방향 고정 — 스냅 축 또는 자유 각도). `m`으로 엣지 스냅을 켜면 측정점이 주변 밝기 경계에 자동 스냅되고 첫 점은 십자 마커로 스냅 위치를 미리 보여준다. `Alt`를 누르는 동안 스냅 일시 해제 |
| `m` | ruler 엣지 스냅 켜기/끄기 — 측정점이 커서 주변에서 가장 뚜렷한 밝기 경계로 서브픽셀 정밀도로 붙는다 (기본 꺼짐, 평탄한 영역에서는 동작 안 함) |
| `d` | 도형 그리기 — 대화상자에서 도형 종류(box/ellipse/line)와 외곽선(use·색)·검은 테두리(black halo)·박스/타원 채우기(use·색·opaque)·선 스타일(굵기 1~8px, solid/dashed/dotted — 라인·윤곽선 공통)을 고르면 그리기 모드가 시작된다 (`Shift` 클릭은 정사각형·정원·수평/수직/45°, 종료는 `Esc`). 윤곽선·채우기 중 하나는 항상 켜져 있으며 텍스트 스타일은 `t` 대화상자에서 별도 선택 |
| `t` | 텍스트 주석 — 클릭 위치에 입력, 폰트 크기·글자 색·배경(사용/색/불투명)을 대화상자에서 직접 선택 (실시간 미리보기, 도형 설정과 별개) |
| `n` | 노트 — 이미지 전체에 대한 메모 입력/수정. 위치·크기·색상이 없고 좌상단(키 안내 아래)에 정보 칩으로 고정 표시된다. 내용을 비우고 OK 하면 제거, `Ctrl+S`로 주석과 함께 저장 |
| `s` / `Shift+s` | 주석 선택 순회 (최신부터 / 반대 방향) — 선택 중에는 방향키가 주석을 이동(`Shift`+방향키는 10px)하고 `e`가 수정(도형은 크기 조절 앵커 순회, 텍스트는 입력 대화상자 재열기), `Delete`/`BackSpace`가 삭제, `Esc`가 해제한다. 이동·수정·삭제도 `u`/`y`로 되돌릴 수 있다 |
| `u` / `y` | 주석 실행 취소 / 다시 실행 — `u`가 최신 주석부터 하나씩 지우고(파일에서 복원된 주석 포함) `y`가 되살린다. 새 주석을 그리면 다시 실행 이력은 사라진다 |
| `Ctrl+C` | 보이는 화면을 클립보드에 복사 — 정보 표시(키 안내·상태·경로·legend·외곽선·노트)는 자동으로 빼고 주석·측정선은 화면 그대로(`Tab` 토글 반영) 캡처. 뷰어가 selection을 직접 서비스하므로(클립보드 매니저에 이관하면 원격 X 환경에서 이미지 타깃이 유실될 수 있음) 복사본은 뷰어가 떠 있는 동안 유효하다 |
| `Ctrl+Shift+C` | 현재 보고 있는 이미지의 전체 경로를 텍스트로 클립보드에 복사 (스택은 화면에 표시 중인 레벨 이미지의 경로). 이미지 복사와 마찬가지로 뷰어가 떠 있는 동안 유효하다 |
| `Ctrl+S` | 주석·PPU 메타데이터 저장 — 자동 저장이 없으므로 이 명시적 저장이 유일한 보존 수단이다. PNG는 원본 파일 안에 `iTXt` 청크로 내장하고(픽셀은 그대로, 다른 뷰어에서도 평범한 PNG로 열리며 파일을 복사하면 주석이 함께 이동), 그 외 포맷은 `<이미지경로>.fe` 사이드카에 저장한다(읽기 전용이면 `~/.cache/flateyes/`). 주석이 없는 상태에서 누르면 저장된 메타데이터를 제거한다 |
| `p` | PPU(pixels per unit) 입력 — `Ctrl+S`로 저장하면 파일별로 복원된다 (스택 모드에서는 manifest 값 사용) |
| `[` / `]` | 스택 이전/다음 배율 레벨로 이동 |
| `F1` / 우클릭 메뉴 | About (프로그램 정보) |
| `q` | 종료 — 저장 안 된 주석이 있으면(창 제목 파일명 앞 `*` 표시) Save/Discard/Cancel 대화상자로 묻는다. 측정(ruler)만 바뀐 상태는 묻지 않고 종료한다. 창 닫기(X)도 동일 (`Esc`는 종료하지 않는다 — 선택·측정·도구 모드를 차례로 빠져나오고, 더 빠져나올 것이 없으면서 브라우저에서 연 이미지라면 브라우저로 돌아간다) |

## 지원 포맷

GdkPixbuf가 지원하는 모든 포맷: JPEG(EXIF 회전 자동 적용), PNG, GIF(애니메이션 재생), BMP, TIFF, ICO, XPM 등 — eog와 동일한 디코더를 쓰므로 eog에서 열리던 이미지는 모두 열린다.

## 참고 사항

- 창을 닫으면(`q`) 인스턴스가 종료되며, 다음 실행 시 새 창이 뜬다.
- 윈도우 매니저의 포커스 강탈 방지 정책 때문에 이미지 갱신 시 창이 최상단으로 올라오지 않는 환경에서는 작업 표시줄 깜빡임(urgency hint)으로 대신 알린다.
- `:1`과 `:1.0`은 같은 디스플레이로 취급한다 (스크린 번호 무시).
