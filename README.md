# tdviewer

폐쇄망 리눅스 환경용 초경량 이미지 뷰어 (단일 파일 Python 스크립트).

eog를 대체하기 위해 만들어졌으며, eog의 두 가지 문제를 동시에 해결한다:

| | eog | tdviewer |
|---|---|---|
| 같은 디스플레이에서 반복 실행 | 창이 계속 늘어남 (또는 `--single-instance` 시 아래 문제 발생) | **기존 창 하나가 이미지를 교체 표시** |
| 여러 사용자 디스플레이에 동시 표시 | 단일 인스턴스 모드에서는 불가 | **디스플레이마다 독립 인스턴스로 동시 표시 가능** |

## 동작 원리

- 인스턴스 식별 키 = `(실행 uid, $DISPLAY)`. 이 키로 리눅스 abstract namespace 유닉스 소켓을 생성한다 (디스크에 파일을 남기지 않으므로 stale socket 문제 없음).
- 같은 디스플레이에서 다시 실행하면: 기존 인스턴스 소켓에 이미지 경로만 전달하고 **즉시 종료** (GTK도 로드하지 않아 수십 ms 수준). 기존 창이 새 이미지로 교체되고 앞으로 올라온다.
- 디스플레이가 다르면 소켓 이름이 달라지므로 각 사용자 디스플레이마다 창이 하나씩 독립적으로 뜬다.

## 요구 사항

추가 패키지 설치 불필요 — GNOME(eog)이 있는 시스템에 기본 포함된 것만 사용:

- Python 3
- PyGObject + GTK3 (`python3-gobject`, RHEL/CentOS GNOME 환경 기본 포함)

대상 서버에서 사전 확인:

```sh
python3 -c 'import gi; gi.require_version("Gtk", "3.0"); print("OK")'
```

`OK`가 출력되면 바로 사용 가능. (실패한다면 환경 정보를 알려주면 다른 방식으로 포팅 가능)

## 설치

파일 하나만 복사하면 된다:

```sh
cp tdviewer.py /usr/local/bin/tdviewer   # 경로는 자유
chmod +x /usr/local/bin/tdviewer
```

## 사용법

```sh
DISPLAY=:1 tdviewer /path/to/image.jpg
```

- 첫 실행: 해당 디스플레이에 창을 열고 계속 떠 있음 (백그라운드 상주).
- 이후 실행: 같은 창에 이미지를 교체 표시하고 즉시 종료 (exit 0).
- 이미지 로드 실패 시: 오류를 stderr로 출력하고 exit 1 (기존 창의 이미지는 유지).

### 런처(기존 eog 실행 프로그램)에서 호출

eog를 호출하던 자리에 그대로 치환하면 된다. 첫 실행 프로세스는 창이 닫힐 때까지 살아 있으므로, 런처가 종료를 기다리지 않도록 분리 실행 권장:

```sh
sudo -u tduser DISPLAY=:1 XAUTHORITY=/home/tduser/.Xauthority \
    setsid /usr/local/bin/tdviewer /data/images/alarm.png >/dev/null 2>&1 &
```

같은 명령을 몇 번을 실행해도 창은 디스플레이당 1개만 유지된다. 두 번째 실행부터는 이미 떠 있는 인스턴스에 경로만 넘기고 바로 끝나므로 `&` 없이 호출해도 무방하다.

## 키 조작

| 키 | 동작 |
|---|---|
| `+` / `-` (Ctrl+휠) | 확대 / 축소 (최대 200%) |
| `1` | 원본 크기 (100%) |
| `f` 또는 `0` | 창 크기에 맞춤 (기본값, 축소만 하고 확대는 안 함) |
| `F11` | 전체 화면 토글 |
| 방향키 / 휠 / 마우스 드래그 | 스크롤(패닝) |
| `q` / `Esc` | 종료 |

## 지원 포맷

GdkPixbuf가 지원하는 모든 포맷: JPEG(EXIF 회전 자동 적용), PNG, GIF(애니메이션 재생), BMP, TIFF, ICO, XPM 등 — eog와 동일한 디코더를 쓰므로 eog에서 열리던 이미지는 모두 열린다.

## 참고 사항

- 창을 닫으면(`q`/`Esc`) 인스턴스가 종료되며, 다음 실행 시 새 창이 뜬다.
- 윈도우 매니저의 포커스 강탈 방지 정책 때문에 이미지 갱신 시 창이 최상단으로 올라오지 않는 환경에서는 작업 표시줄 깜빡임(urgency hint)으로 대신 알린다.
- `:1`과 `:1.0`은 같은 디스플레이로 취급한다 (스크린 번호 무시).
